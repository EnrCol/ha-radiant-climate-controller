# v0.2 - Base osservatore predittivo

## Scopo

La v0.2 non comanda ancora nulla.

Serve a creare una base dati affidabile per capire se la centralina sta scegliendo un target mandata sensato prima di collegarla a ESPHome.

## Input usati

### Stanze

- Cucina
- Soggiorno
- Bagno
- Studio
- Camera
- Camera Ricky

Per ogni stanza vengono letti:

- temperatura stanza;
- umidità stanza;
- punto rugiada stanza, se esistente;
- input boolean sicurezza rugiada;
- termostato associato.

### Zone deumidifica

- Zona giorno: cucina, soggiorno
- Zona notte: bagno, studio, camera, camera Ricky

Per ogni zona vengono letti:

- richiesta deumidificazione;
- consenso deumidificatore;
- temperatura media;
- umidità media;
- soglia umidità.

### Sicurezze globali

- `input_select.modalita_stagionale`
- `input_number.rugiada_temperatura_delta_sicurezza`
- `sensor.punto_rugiada_massimo_casa`
- `binary_sensor.standby_rugiada`
- `binary_sensor.standby_porte`
- `binary_sensor.standby_attivo`
- `binary_sensor.acs_blocco_valvola_miscelatrice`

## Sensori creati

La v0.2 espone sensori per:

- temperatura massima casa;
- umidità massima casa;
- punto rugiada massimo casa;
- stanza più calda;
- temperatura stanza più calda;
- trend temperatura casa;
- stanza rugiada critica;
- zona rugiada critica;
- delta rugiada critico;
- stato rischio rugiada;
- target mandata comfort;
- target mandata sicuro;
- target mandata consigliato;
- azione consigliata;
- motivo target;
- motivo azione.

## Strategia target

La centralina sceglie un target comfort con logica sia reattiva sia anticipata.

| Stato | Reattivo | Anticipo se trend positivo | Target |
|---|---:|---:|---:|
| Mantenimento | sotto soglia | stabile | 19.2°C |
| Normale | >= 25.2°C | >= 24.8°C | 19.0°C |
| Spinto | >= 25.7°C | >= 25.3°C | 18.5°C |
| Recupero | >= 26.2°C | >= 25.8°C | 18.0°C |

Poi applica il limite rugiada:

```text
target_consigliato = max(target_comfort, punto_rugiada_massimo_casa + delta_sicurezza)
```

## Azione consigliata

La centralina produce solo una stringa, non un comando.

Possibili azioni:

- `non_estate`
- `blocco_acs`
- `standby_porte`
- `protezione_globale_mandata`
- `deumidificare_zona`
- `protezione_locale_stanza`
- `anticipo_raffrescamento`
- `mantenimento`

## Cosa validare

Prima di collegare il target a ESPHome bisogna osservare:

1. se la stanza più calda è corretta;
2. se il trend non è troppo rumoroso;
3. se il target consigliato si muove prima che la casa sia già calda;
4. se il limite rugiada non alza la mandata troppo spesso;
5. se l'azione consigliata è coerente con la situazione reale.
