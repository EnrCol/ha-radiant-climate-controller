# v0.3 - Configurazione da UI

## Scopo

La v0.3 rende configurabile la centralina senza modificare codice.

La centralina resta osservativa: non comanda ancora ESPHome, termostati, testine o deumidificatori.

## Entità number aggiunte

### Soglie reattive

- Soglia normale
- Soglia spinto
- Soglia recupero

Queste soglie determinano lo stato quando la temperatura massima casa ha già superato il limite.

### Soglie anticipo

- Anticipo normale
- Anticipo spinto
- Anticipo recupero

Queste soglie lavorano insieme al trend temperatura casa.

Esempio:

```text
Temperatura massima casa >= Anticipo spinto
AND
Trend temperatura casa >= Trend spinto
→ stato spinto anticipato
```

### Trend minimi

- Trend normale
- Trend spinto
- Trend recupero

Unità: °C/h.

### Target mandata

- Target mantenimento
- Target normale
- Target spinto
- Target recupero

Questi valori generano il target mandata comfort.

Il target finale resta sempre protetto da rugiada:

```text
target_consigliato = max(target_comfort, target_sicuro)
```

### Margine rugiada

Il margine rugiada viene usato per calcolare:

```text
target_sicuro = punto_rugiada_massimo_casa + margine
```

Nota: se questo number non è stato modificato, la centralina continua a leggere l'helper Home Assistant esistente:

```text
input_number.rugiada_temperatura_delta_sicurezza
```

## Entità select aggiunta

### Stato manuale

Opzioni:

- auto
- mantenimento
- normale
- spinto
- recupero

In `auto` la centralina decide da sola.

Negli altri stati, il target comfort viene forzato al target corrispondente.

## Uso consigliato

Durante i test lasciare `Stato manuale` su:

```text
auto
```

Usare il manuale solo per verificare rapidamente come cambia il target consigliato.
