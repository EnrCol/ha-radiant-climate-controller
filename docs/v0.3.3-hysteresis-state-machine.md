# v0.3.3 - State machine con isteresi

## Scopo

La v0.3.3 stabilizza lo stato radiante prima del collegamento a ESPHome.

Il radiante ha inerzia alta. Lo stato non deve cambiare continuamente quando la temperatura oscilla vicino a una soglia.

## Principio

La centralina ora mantiene memoria dello stato automatico precedente.

Se lo stato calcolato scenderebbe a un livello piu basso, la centralina mantiene lo stato precedente finche la temperatura non scende sotto una soglia di uscita.

## Isteresi

Isteresi fissa:

```text
0.3 C
```

Esempio con soglie standard:

```text
recupero entra sopra 26.2 C, esce sotto 25.9 C
spinto entra sopra 25.7 C, esce sotto 25.4 C
normale entra sopra 25.2 C, esce sotto 24.9 C
```

## Uso del trend

Il trend positivo continua a servire per anticipare l ingresso negli stati piu forti.

Il trend negativo non forza da solo l uscita immediata da recupero, spinto o normale.

## Esempio

Se la casa e a 26.7 C con trend 30m negativo, la centralina puo restare in recupero perche la temperatura e ancora sopra la soglia di uscita.

Quando la temperatura scende sotto la soglia di uscita recupero, la centralina puo scalare a spinto.

## Nota operativa

La memoria dello stato e in RAM. Dopo riavvio Home Assistant, la state machine riparte dallo stato calcolato al primo aggiornamento.
